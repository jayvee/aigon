The Amplification Dashboard is part of [Aigon Pro](/pro). It builds on the telemetry data that the free Reports tab already collects, adding deeper quality analysis, trend tracking, and AI-powered coaching to help you get more value from your multi-agent workflow.

> **Prerequisite:** Aigon Pro must be installed and linked. See the [Pro page](/pro) for setup instructions.

## Quality metrics leaderboard

The leaderboard ranks agents by code quality, not just throughput. Each agent gets a row with these metrics:

| Metric | What it measures | Better is... |
|--------|-----------------|--------------|
| **First-pass rate** | % of features that passed evaluation on the first attempt | Higher |
| **Commits per feature** | Median commits needed to complete a feature | Lower |
| **Rework ratio** | % of commits that are fixes (`fix:`, `fixup`, `bugfix`) | Lower |
| **Cycle time** | Median wall-clock time from start to close | Lower |
| **Win rate** | % of Fleet evaluations won (Fleet mode only) | Higher |

Use the leaderboard to decide which agents to assign to which roles. An agent with a high first-pass rate but slow cycle time might be better as an evaluator than an implementer.

## Trend charts

Three time-series charts track whether your workflow is improving:

- **Cycle Time Trend** — is development getting faster or slower?
- **Rework Trend** — are agents producing cleaner code over time?
- **Cost per Feature** — is each feature getting cheaper to deliver?

Charts share a daily/weekly/monthly granularity toggle. Hover for exact values.

## AI insights & coaching

The insights panel analyses your recent workflow data and surfaces:

- **Observations** — patterns the AI has detected (e.g. "Gemini's rework ratio has increased 40% over the last two weeks")
- **Recommendations** — actionable suggestions (e.g. "Consider using Claude for evaluation instead of implementation — its first-pass rate is 15% higher when reviewing code")
- **Cost optimization** — opportunities to reduce spend without sacrificing quality (e.g. "Switching implementation to Sonnet from Opus would save ~$12/feature with no measurable quality drop based on your last 20 features")

Insights are regenerated each time you open the Amplification tab, using your latest telemetry data.

## How data flows

```
Agent sessions → Telemetry parsers → .aigon/telemetry/ → Reports tab (free)
                                                        ↘ Amplification (Pro)
```

Amplification reads the same normalized telemetry that the free Reports tab uses. No additional data collection is required — if Reports shows data, Amplification can analyse it.

## Setting up Pro

1. Install the Pro package:

```bash
cd ~/src/aigon-pro
npm link
cd ~/src/aigon
npm link @aigon/pro
```

2. Verify Pro is detected:

```bash
aigon doctor
```

Look for the "Aigon Pro" line in the output.

3. Restart the dashboard to load the Amplification tab:

```bash
aigon server restart
```

## When Amplification is most useful

Amplification shines when you have enough data to spot trends:

- **After 10+ features** — the leaderboard becomes statistically meaningful
- **With 2+ agents** — comparisons between agents reveal strengths and weaknesses
- **Over time** — trend charts need at least 2-3 weeks of data to show direction

For new projects with few features, the free Reports tab provides everything you need.
