---
title: Scheduled kickoffs
description: Run `feature-autonomous-start` or `research-start` later, at a wall time you choose, with explicit timezones.
---

Scheduled kickoffs let you **queue** a deferred run instead of starting work immediately. The dashboard server runs a periodic poller; when `runAt` is in the past, Aigon spawns the same CLI entrypoints you would use by hand (`feature-autonomous-start` or `research-start`).

## Where jobs live

Per repository, jobs are stored in **`.aigon/state/scheduled-kickoffs.json`**. Each job has a stable `jobId`, a `runAt` ISO timestamp, a `kind`, a numeric `entityId`, and a normalized `payload` that matches the CLI flags described below.

## Time and timezones

`runAt` must be a full **ISO 8601** datetime that includes an explicit timezone:

- UTC suffix `Z`, e.g. `2026-04-26T01:10:00Z`
- Or a numeric offset, e.g. `2026-04-26T01:10:00+10:00` or `2026-04-26T01:10:00-07:00`

Strings **without** a zone (naive “local” times) are rejected. Always encode the zone you mean — especially when coordinating across regions.

## Catch-up if the server was down

If the dashboard is not running at `runAt`, the job stays **pending** until the next poll **after** the server starts. When `runAt <= now`, the job fires once (no retroactive missed-window storm for old pending rows beyond normal due processing).

## CLI

```bash
# Feature — same options as feature-autonomous-start (agents + flags)
aigon schedule add feature_autonomous <id> --run-at=<iso8601> <agents...> \
  [--stop-after=implement|eval|review|close] \
  [--eval-agent=] [--review-agent=] \
  [--models=] [--efforts=] [--review-model=] [--review-effort=] [--workflow=] \
  [--repo=<path>]

# Research — same shape as research-start
aigon schedule add research_start <id> --run-at=<iso8601> [agents...] [--background|--foreground] [--repo=<path>]

aigon schedule list [--all] [--repo=<path>]
aigon schedule cancel <jobId> [--repo=<path>]
```

With **multiple repos** registered for the dashboard, `--repo=` is required when the CLI cannot infer a single target (same resolution rules as other multi-repo commands).

## Dashboard

- **Settings → Schedule** lists jobs for the selected repository (or all repos), optional **Show fired / cancelled / failed**, and **Cancel** for pending rows.
- On **backlog** feature and research cards, open the **⋯** menu and choose **Schedule** to open a form aligned with the `schedule add` inputs (including workflow slug, autonomous triplets, stop-after, eval/review agents, and research background/foreground).

Scheduling is only offered when a **workflow snapshot** already exists for that row (same guard as `addJob` — snapshotless backlog seeds still show **Start**, but not **Schedule**, until engine state exists).

## Kinds

| Kind | Runs |
|------|------|
| `feature_autonomous` | `aigon feature-autonomous-start …` with your payload |
| `research_start` | `aigon research-start …` with your payload |

Validation and argv construction are shared with the interactive dashboard run path so scheduled runs stay aligned with manual starts.
