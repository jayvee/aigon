---
title: GitHub PR Workflow
description: Use GitHub pull requests for code review before closing features in Aigon.
---

Aigon can integrate with GitHub pull requests so that `feature-close` is aware of PR state. This is entirely optional — if you don't create a PR, Aigon closes features locally as it always has.

## How it works

When you run `feature-close` on a repo with a GitHub origin and the `gh` CLI installed, Aigon automatically checks for a PR on the feature branch:

| PR state | What Aigon does |
|----------|----------------|
| **No PR exists** | Normal local close (merge branch, clean up) |
| **PR is open** | Blocks close with a message — merge or close the PR first |
| **PR is draft** | Blocks close — publish or close the draft first |
| **PR is merged** | Syncs local main from origin and finishes the close |

No configuration is required. The gate activates automatically when all three conditions are met:
1. Your repo's `origin` remote points to GitHub
2. The `gh` CLI is installed and authenticated
3. A PR exists for the feature branch targeting your default branch

If `gh` is missing or not authenticated, Aigon silently falls back to local close.

## Recommended workflow

You'll typically do this from inside a feature worktree — a tmux session that Aigon created when you ran `feature-start` or `feature-open`. The slash commands auto-detect your feature ID and agent from the worktree branch, so you don't need to pass arguments.

### 1. Implement the feature

Use any Aigon mode (Drive, Fleet, Autopilot) to build the feature as normal.

### 2. Push the branch

From your agent session in the worktree:

```
/aigon:feature-push
```

Or from the CLI (main repo or worktree):

```bash
aigon feature-push        # auto-detects from worktree branch
aigon feature-push 07     # explicit ID from main repo
```

This pushes the feature branch to `origin` with upstream tracking.

### 3. Create a pull request

Create the PR on GitHub — through the web UI, `gh pr create`, or however you prefer. Aigon does not create PRs for you.

### 4. Review and merge

Get reviews, iterate, and merge the PR on GitHub when ready.

### 5. Close the feature

From the agent session:

```
/aigon:feature-close
```

Or from the CLI:

```bash
aigon feature-close 07
```

If the PR is merged, Aigon syncs your local main branch from origin and completes the close (telemetry, spec move, cleanup). If the PR is still open, Aigon blocks and tells you what to do.

## Without PRs

If you never push or create a PR, nothing changes. `feature-close` works exactly as before — local merge, cleanup, done. The PR gate only activates when it detects a PR exists for your branch.

## Limitations (v1)

- Aigon does not create PRs — you do that yourself
- Only GitHub is supported (GitLab, Bitbucket are not checked)
- If `gh` auth fails, the gate silently degrades to local close rather than blocking
- No dashboard visibility of PR state (coming in Aigon Pro)
