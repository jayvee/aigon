## Synopsis

```bash
aigon dashboard <start|restart|stop|status|open|add|remove|list|tile>
```

## Description

The Aigon Dashboard is a browser-based control centre that gives you a visual pipeline view of features, research, and feedback across all your registered repos. Monitor agent progress, launch features, run evaluations, and merge winners — all from one place.

For a full guide on using the dashboard, see [Dashboard Guide](/docs/guides/dashboard).

![Dashboard Pipeline view](/img/aigon-dashboard-kanban.png)

## Subcommands

### Server lifecycle

```bash
aigon dashboard start          # Start the dashboard server (port 4100)
aigon dashboard restart        # Stop and restart
aigon dashboard stop           # Stop the server
aigon dashboard status         # Check health and see waiting agents
aigon dashboard open           # Open in browser (http://aigon.localhost)
```

### Repo management

```bash
aigon dashboard add [path]     # Register a repo (default: current directory)
aigon dashboard remove [path]  # Unregister a repo
aigon dashboard list           # Show registered repos
```

### Window management

```bash
aigon dashboard tile           # Tile all iTerm2 agent windows into a grid
```

## Dashboard tabs

| Tab | What it shows |
|-----|---------------|
| **Monitor** | All agent sessions across repos — running, attached, or ended |
| **Pipeline** | Kanban board — drag features through inbox → backlog → in-progress → done |
| **Sessions** | Tmux session management — attach, kill orphans, tile windows |
| **Reports** | Feature throughput, cycle time trends, agent performance |
| **Insights** | AI workflow insights and coaching (Pro) |
| **Logs** | Live agent output and command history |
| **Console** | Interactive terminal for running Aigon commands |
| **Settings** | Dashboard and agent configuration |
| **Config** | View and edit project/global config |

## How it works

The dashboard runs as a local HTTP server on port 4100. It polls registered repos every 10 seconds, reading state from `.aigon/state/` manifests and `docs/specs/` folders. Changes made via the CLI are immediately visible in the dashboard, and vice versa.

- **Proxy URL**: `http://aigon.localhost` (requires `aigon proxy install`)
- **Direct URL**: `http://localhost:4100`
- **Remote access**: Binds to `0.0.0.0` — accessible from any device on your network at `http://<your-ip>:4100`

## Examples

```bash
# Quick start
aigon dashboard start
aigon dashboard add ~/src/my-project
aigon dashboard open

# Register multiple repos
aigon dashboard add ~/src/project-a
aigon dashboard add ~/src/project-b

# Check what's running
aigon dashboard status
```
