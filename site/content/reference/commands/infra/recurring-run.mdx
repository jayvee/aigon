---
title: recurring-run
description: Instantiate any recurring feature templates that are due for the current ISO week or quarter.
---

Check all templates in `docs/specs/recurring/` and create feature instances for any that are due and don't already have an open instance.

## Usage

```bash
aigon recurring-run
```

No arguments. Safe to run at any time — will not create duplicates.

## What it does

For each `.md` file in `docs/specs/recurring/`:

1. Checks whether an open instance (inbox, backlog, in-progress, or in-evaluation) already exists for this week/quarter using the `recurring_slug` frontmatter field
2. Checks `.aigon/recurring-state.json` to see whether an instance was already created this period
3. If neither check trips, instantiates the template: renders `{{YYYY-WW}}` placeholders, assigns an ID, and moves directly to backlog
4. Commits the new spec file with `chore: recurring feature <ID> - <name>`

The background server also calls this automatically on startup and every 24 hours.

## Example output

```

[recurring] weekly-agent-matrix-benchmark: created and prioritised → #381

📅 Week: 2026-W17
📋 Templates found: 3
✅ Created: 1
```

## When to run manually

- After adding a new recurring template and wanting it to fire immediately
- After deleting `.aigon/recurring-state.json` to force a re-run
- When the dashboard server was offline during a week transition
