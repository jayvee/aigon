## Synopsis

```bash
aigon feature-autonomous-start <feature-id> [agents...] [--eval-agent=<agent>] [--stop-after=implement|eval|close]
aigon feature-autonomous-start status <feature-id>
```

## Shortcuts

- `/aigon:feature-autonomous-start` (slash command)

## Description

Starts the **AutoConductor** — a background tmux session that monitors the workflow engine and triggers the next lifecycle stage automatically. When all agents submit, it spawns the eval agent (Fleet) or closes the feature directly (solo).

## Flags

| Flag | Description |
|------|-------------|
| `--eval-agent=<agent>` | Agent to run evaluation (Fleet only). Defaults to first implementation agent. |
| `--stop-after=<value>` | Where to stop: `implement`, `eval`, or `close` (default: `close`) |

## Modes

| Invocation | Mode | Behaviour |
|-----------|------|-----------|
| `feature-autonomous-start 42 cc` | Solo | Implement → close automatically |
| `feature-autonomous-start 42 cc gg` | Fleet (close) | Implement → auto-eval → auto-close winner |
| `feature-autonomous-start 42 cc gg --stop-after=eval` | Fleet (eval) | Implement → auto-eval → pause for winner selection |
| `feature-autonomous-start 42 cc --stop-after=implement` | Implement only | Implement → pause |

## Usage

```bash
# Solo — fully hands-off through close
aigon feature-autonomous-start 42 cc

# Fleet — two agents, auto eval and close
aigon feature-autonomous-start 42 cc gg --eval-agent=cc

# Fleet — stop after eval, pick winner manually
aigon feature-autonomous-start 42 cc gg --eval-agent=cc --stop-after=eval

# Check AutoConductor status
aigon feature-autonomous-start status 42
```

### From the dashboard

Click **Start Autonomously** on any backlog feature card. Choose implementation agents, evaluator, and stop-after in the modal.

## AutoConductor session

The AutoConductor runs as a detached tmux session named `<repo>-f<id>-auto`. Poll interval is 30 seconds.

To attach:

```bash
tmux attach -t brewboard-f42-auto
```

The AutoConductor prints verbose poll logs:

```
[08:33:45] state=implementing agents=[cc=running, gg=running] allReady=false ...
[08:34:15] state=implementing agents=[cc=ready, gg=running] allReady=false ...
[08:34:45] state=implementing agents=[cc=ready, gg=ready] allReady=true ...
🚀 Triggering eval: brewboard-f42-eval-cc-...
✅ Started eval session: brewboard-f42-eval-cc-...
  [eval-start 1/10] state=evaluating evalSession running=true
✅ Evaluation started. Waiting for eval agent to complete and close...
  [eval-close 1/120] state=evaluating evalSession running=true
  [eval-close 2/120] state=evaluating evalSession running=true
  ...
✅ Feature closed autonomously. AutoConductor finished.
```

## Resilience

If the AutoConductor dies mid-run, the feature continues. Complete it manually:

```bash
aigon feature-eval 42             # if not yet evaluated
aigon feature-close 42 cc         # merge the winner
```
