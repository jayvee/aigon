2 zones (title + status). State, agent, duration, and CTA in one scannable line.
Key principles: 1 — One status row, not two.   2 — Liveness dot lives inside the agent token.   3 — Detail line (italic, dim) only when there is actionable context (message, next step hint).   4 — Primary CTA is always right-aligned in the status row — no separate action bar.   5 — Duration only for active/just-finished states; omit for static ones.

4 · Design Decisions Summary

WasNow
headline banner block left-border on status row
separate agent row below banner agent token inside status row
separate action-button row CTAs right-aligned in status row
awaiting-badge chip (separate DOM) label overrides to "Needs you" + detail line
idle-ladder chip (separate DOM) stale/dead dot + detail line + label override
liveness dot floating near title dot embedded in agent token ⬤ CC
Drive/Fleet badge on every card mode shown at column/toolbar level only
Card height: pre-start = 1 row · active (no detail) = 1 row · active with detail = 2 rows · fleet = 1 summary + N agent rows.
Max card height is bounded: no state needs more than title + status + detail + (N fleet rows).
