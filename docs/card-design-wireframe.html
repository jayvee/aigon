Pipeline Card — Design Wireframe v2

Precise vocabulary + all states · branch design/card-redesign · 2026-05-06

1 · Vocabulary — what each label actually means

Every label must answer the question "what happened?" or "what is happening?" with no ambiguity. "Complete" and "Done" both fail that test — they say nothing specific about what is complete or done.

Old label New label Precise meaning
COMPLETE Implemented The implementation agent finished writing code and pushed the branch. Nothing else.
REVISION DONE Revised The implementation agent addressed review feedback and pushed a new revision.
REVIEW DONE Code reviewed A reviewer agent read the code and wrote findings. The code has been reviewed.
SPEC REVIEW DONE Spec reviewed A reviewer agent read the spec and wrote findings. The spec has been reviewed.
RESEARCH DONE Research ready A research agent produced findings. The findings exist and are ready to evaluate.
Done (terminal) Closed aigon feature-close ran. The worktree was merged or discarded. The lifecycle is over.
Needs you / Waiting Needs input The agent hit a decision it can't make alone and stopped to ask. A human must respond.
awaiting review (detail) → needs code review Imperative. Names the missing action. "Awaiting" is passive and says nothing about what is awaited.
apply or reject (detail) → apply or reject findings "Findings" clarifies what is being applied/rejected. Slightly more specific.
All complete (fleet) All implemented Each agent has finished implementing. "Complete" is ambiguous at the feature level.
Finished (unconfirmed) Unconfirmed The agent's session ended without a completion signal. State is unknown — human must confirm.
Rule: If you can swap the label with "Complete" and lose no information — the label is wrong. Every label must name a specific act or specific resource: Implemented, Code reviewed, Research ready, Closed.

3 · Card anatomy

[ID] [Feature name] [▾]
[icon] [State label] · ⬤[Agent] · [Duration] [👁] [CTA] [···]
[detail — only when there is an actionable message, blocked-by, or error context]

Two rows maximum for most states. The left-border colour encodes urgency at a glance (green = running, teal = handoff, amber = waiting, red = error, grey = static). The liveness dot (⬤) is embedded in the agent token — its colour shows session health: green (alive), amber (stale >5 min silent), red (dead / session gone). The detail line appears only when there is something specific to communicate.
