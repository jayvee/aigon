# Feature: onboarding-prereq-detectors

## Summary

Extract prerequisite detection into a shared `lib/onboarding/detectors.js` module that exposes a per-item `{ id, label, check(), install(), verify() }` contract. The wizard, `aigon doctor`, and `install-agent` all need to detect the same set of tools (Node, git, gh, tmux, agent CLIs) but currently consume a batch-only API (`runPrerequisiteChecks()` in `lib/prerequisite-checks.js`) that cannot drive spinner-per-step wizard flows. This feature builds the per-item primitives and updates existing consumers to use them, leaving the batch API intact for backward compatibility.

## User Stories

- [ ] As a wizard step, I can call `detector.check()` inside a `spinner()`, then conditionally call `detector.install()` under a second `spinner()`, then `detector.verify()` to confirm — without knowing how each tool is detected or installed.
- [ ] As `aigon doctor`, I can iterate the detector list and get the same results as today, just sourced from the new module rather than inline logic.
- [ ] As `install-agent`, I can call an agent's detector directly to verify a binary is on PATH after install.

## Acceptance Criteria

- [ ] `lib/onboarding/detectors.js` exports `getDetectors()` returning an ordered array of detector objects, each with: `{ id, label, required: boolean, check(): Promise<{found, version?}>, install(): Promise<{ok, output?}>, verify(): Promise<{found, version?}> }`.
- [ ] Detectors covered: `node` (hard, ≥18), `npm` (hard), `git` (hard), `gh` (soft), `tmux` (soft).
- [ ] Agent CLI detectors are generated from the agent registry: `getAgentDetectors()` returns one detector per registered agent, sourced from `agent.cli.command` (binary) and `agent.installHint` / a new `agent.installCommand` field added to each `templates/agents/*.json`.
- [ ] `lib/prerequisite-checks.js` `runPrerequisiteChecks()` is refactored to delegate to `getDetectors()` + `getAgentDetectors()` instead of duplicating logic — existing output shape (errors/warnings/infos/passed) is preserved exactly.
- [ ] `install()` for tmux, gh: runs the platform-appropriate install command (`brew install tmux` on macOS, `sudo apt install tmux` on Linux); wraps output and exit code; returns `{ ok: boolean, output: string }`. If no install command is available for the platform, returns `{ ok: false, output: 'Manual install required: <hint>' }`.
- [ ] `install()` for agent CLIs: runs the `installCommand` from the agent template JSON; captures stdout/stderr.
- [ ] `verify()` is always `check()` re-run after install — not a separate implementation.
- [ ] All detector functions return plain objects (no throws on missing binaries); callers decide how to surface failures.
- [ ] `npm test` passes with no regressions to doctor or install-agent paths.
- [ ] New `templates/agents/*.json` field `installCommand` (machine-executable, e.g. `npm i -g @anthropic-ai/claude-code`) added for cc, gg, cx; cu marked as `"installCommand": null` (GUI download only).

## Technical Approach

**New module: `lib/onboarding/detectors.js`**

Each detector follows this shape:
```js
{
  id: 'tmux',
  label: 'tmux',
  required: false,
  async check() { /* isBinaryAvailable + version probe */ },
  async install() { /* platform-aware shell-out, capture output */ },
  async verify() { return this.check(); }
}
```

`check()` and `verify()` are async wrappers over the existing sync `isBinaryAvailable()` in `lib/security.js` (no new detection logic, just reshape).

`install()` shells out using `execFileSync`/`spawnSync` with `stdio: 'pipe'`, captures combined output, returns `{ ok, output }`. Never throws.

Agent detectors are generated by iterating `agentRegistry.getAllAgents()` and using `agent.cli.command` for the binary and `agent.installCommand` (new field) for the install shell string.

**`templates/agents/*.json` additions:**
- `installCommand`: machine-executable install string, e.g. `"npm i -g @anthropic-ai/claude-code"` for cc, `null` for cu (GUI only).

**`lib/prerequisite-checks.js` refactor:**
- `runPrerequisiteChecks()` calls `getDetectors()` + `getAgentDetectors()`, runs `check()` on each, and maps `required: true` failures → `errors`, `required: false` failures → `warnings`, passing → `infos`. Output shape is identical to today.

**Do not** create `lib/onboarding/` as a package with its own `package.json` — a plain directory is sufficient.
