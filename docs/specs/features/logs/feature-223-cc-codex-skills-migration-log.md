# Implementation Log: Feature 223 - codex-skills-migration
Agent: cc

## Summary

Migrated `aigon install-agent cx` from the deprecated
`~/.codex/prompts/aigon-*.md` global prompts to project-local Skills
under `.agents/skills/aigon-*/SKILL.md`. The aigon-spawned Codex
launch path (`lib/agent-prompt-resolver.js → resolveCxPromptBody`) is
unchanged — it already inlines the prompt body and never depended on
codex's slash-command discovery. This change is purely about
*interactive* user-typed invocations and removing the deprecated
artifacts.

## Decisions

- **Project-scoped Skills, not user-scoped.** Installing under
  `./.agents/skills/` matches every other agent's project-local
  install pattern and removes the long-standing "global prompts
  shared across projects" complaint. Users can still drop personal
  skills into `~/.agents/skills/` themselves.
- **No alias-skill generation.** Aliases are a slash-command
  shortcut idea — for Codex the equivalent is description-driven
  implicit invocation. Generating `$afd`, `$afs`, etc. as separate
  skills would pollute the implicit-invocation namespace.
- **`.codex/prompt.md` removed entirely, not migrated.** It is a
  static project-context document, not a workflow command, and
  context delivery for Codex now happens through the per-skill
  `description:` fields combined with the existing `AGENTS.md` flow.
- **Inline launch path preserved.** `agent-prompt-resolver.js` still
  inlines the canonical template body for aigon-spawned sessions.
  This makes the migration risk-free for autonomous flows: even if a
  user has zero skills installed, every aigon-launched Codex session
  still receives the full prompt as its initial argument.
- **Doctor shows install paths.** The acceptance criterion required
  that `aigon doctor` no longer reports `~/.codex/prompts/` for cx.
  Doctor previously did not show install paths at all, so a small
  "Agent install paths" section was added that derives the path from
  each agent's `output` config — this is data-driven and stays
  accurate for any future format changes.
