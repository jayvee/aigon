# Feature: conductor-menubar

## Summary

A macOS menubar indicator that shows live agent status across all watched repos, built as an xbar or SwiftBar plugin. Always visible in the menubar: a count of waiting/running agents. Click to expand a menu showing every feature and agent state with copy-pasteable slash commands. Built on top of the conductor-daemon's data — the plugin script reads the same log file front matter directly. Zero new infrastructure.

## User Stories

- [ ] As a developer, I want to glance at my menubar and know at a glance how many agents are waiting for my input
- [ ] As a developer, I want to click the menubar icon and see all features and agent states across all my repos without opening a terminal
- [ ] As a developer, I want to click an item in the menubar menu and have the slash command copied to my clipboard ready to paste

## Acceptance Criteria

- [ ] `aigon conductor menubar-install` generates and installs an xbar/SwiftBar plugin script to `~/Library/Application Support/xbar/plugins/` or `~/.swiftbar/`
- [ ] Plugin script reads `~/.aigon/config.json` for the list of watched repos
- [ ] Plugin reads log file front matter from each repo's in-progress features
- [ ] Menubar title shows a summary: `⚙ 2 waiting` or `⚙ all done` or `⚙ –` (nothing running)
- [ ] Expanded menu shows one section per repo, one line per agent with status indicator
- [ ] Lines with `waiting` status show the appropriate slash command (e.g. `/afd 30`)
- [ ] Clicking a menu item copies the command to clipboard
- [ ] Plugin refreshes every 30 seconds
- [ ] Works with both xbar and SwiftBar (detect which is installed)
- [ ] `aigon conductor menubar-uninstall` removes the plugin

## Validation

```bash
node --check aigon-cli.js
```

## Technical Approach

### Plugin format (xbar/SwiftBar)

Both xbar and SwiftBar execute a script and render its stdout as a menu. The script outputs:

```
⚙ 2 waiting
---
~/src/aigon
#30 board-action-hub
-- cc: waiting | bash="echo '/afd 30' | pbcopy" terminal=false
-- gg: implementing
-- cx: submitted
---
~/src/my-web-app
#12 dark-mode
-- solo: waiting | bash="echo '/afd 12' | pbcopy" terminal=false
```

The plugin script is a Node.js or shell script generated by `aigon conductor menubar-install` and placed in the appropriate plugins directory. It reads log front matter directly — no daemon required, though the two can coexist.

### Install locations

- xbar: `~/Library/Application Support/xbar/plugins/aigon.30s.sh` (`.30s.` = 30 second refresh)
- SwiftBar: `~/.swiftbar/aigon.30s.sh`

### Detection

`aigon conductor menubar-install` checks for xbar or SwiftBar app bundles in `/Applications/` and installs to the right location. If neither found, prints install instructions.

## Dependencies

- Feature: log-status-tracking (required — plugin reads log front matter)
- Feature: conductor-daemon (optional — daemon provides notifications; menubar is independent)
- External: xbar or SwiftBar must be installed by the user

## Out of Scope

- Building a native macOS app (conductor-web-dashboard covers the richer UI need)
- Auto-running slash commands (just copies to clipboard)
- Windows/Linux menubar support

## Open Questions

- xbar vs SwiftBar: SwiftBar is more actively maintained. Should we default to SwiftBar?
