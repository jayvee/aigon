# model-refresh-{{YYYY-MM}}

## Summary

Probe OpenRouter and Gemini for newly available models, auto-include suitable ones into the
Aigon registry, and produce a brief report for user review. Suitable models (✅) are added
automatically. Risky candidates (⚠️ thinking-mode / too-small / too-expensive) are excluded
but listed so the user can decide whether to add them manually.

## Acceptance Criteria

- [ ] Run `aigon model-refresh --non-interactive` from the aigon repo root
- [ ] Capture the output — note how many models were found, included, and excluded per provider
- [ ] Write `docs/reports/model-refresh-{{YYYY-MM}}.md` using the report template below
- [ ] If any models were added, restart the server: `aigon server restart`
- [ ] Commit report (and any updated agent JSON files): `git add docs/reports/model-refresh-{{YYYY-MM}}.md templates/agents/ && git commit -m "chore(model-refresh): {{YYYY-MM}} registry update"`
- [ ] Close the feature (no eval step needed)

## Technical Approach

```bash
# From aigon repo root
aigon model-refresh --non-interactive 2>&1 | tee /tmp/model-refresh-out.txt
```

Parse `/tmp/model-refresh-out.txt` to extract:
- Models auto-included (look for lines starting with `✅ auto-included:`)
- Models auto-excluded (look for lines starting with `⚠️  auto-excluded:`)
- Filtered count (look for `filtered N irrelevant model(s)`)
- Providers that failed (look for `⚠️  ... fetch failed`)

Then write the report and commit.

Write to `docs/reports/model-refresh-{{YYYY-MM}}.md`:

```markdown

| Model | ID | Price (in/out per MTok) |
|-------|----|-------------------------|
| ...   | ...| ...                     |

## Excluded candidates (⚠️)

<!-- Risky models that were not added. User can include manually via `aigon model-refresh`. -->

| Model | ID | Reason |
|-------|----|--------|
| ...   | ...| ...    |

## Filtered (irrelevant modality)

N models filtered automatically: TTS / VL / robotics / computer-use / superseded.

## Provider status

- OpenRouter: OK / fetch failed
- Gemini: OK / GEMINI_API_KEY not set / fetch failed

## Registry size after update

- gg (Gemini): N active models

---

```

## Pre-authorised

- Skip eval step: registry update + report only, no code changes requiring review.
- Write to `docs/reports/model-refresh-{{YYYY-MM}}.md` and update `templates/agents/op.json` / `templates/agents/gg.json`.
- Restart dashboard after agent JSON updates: `aigon server restart`.
- Commit with message `chore(model-refresh): {{YYYY-MM}} registry update`.
